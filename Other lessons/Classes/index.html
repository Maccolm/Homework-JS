Task 15

There are two classes MultChecker (a class for checking the multiplication table - randomly generated numbers that need to be multiplied), AddChecker (a class for checking addition - randomly generated numbers in a given range that need to be added). Both classes send test results to an object of the Hystory class, which stores the test history in an array of objects
Example.
testsList = [
{ firstNum: 1, secondNum: 5, opration:'*', userAnswer: 7, correctAnswer: 5},
{ firstNum: 3, secondNum: 4, opration:'+', userAnswer: 7, correctAnswer: 7},
]
You can create a separate TestData class that describes one such test and will have these fields.
Develop the TestManager class, which using these classes with the help of a timer periodically generates some N tasks (we randomly choose what to survey: addition or multiplication) and conducts a survey. The test results are added to the History object. Make the object look like this only one class could be created. When these N tasks are completed, display the entire story on the screen.
